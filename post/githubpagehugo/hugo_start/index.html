使用githubPage+hugo无成本建立博客

0 前言

环境：ubuntu22.04

  • GitHub Pages 是一个静态站点托管服务，可以直接将页面托管于 GitHub 库或仓库 (repository) 中。

  • Hugo 是一个用 Go 语言编写的静态站点生成器，其优势为速度快。（还有hexo、jekyll）

  • GitHub Pages 是一个静态站点托管服务，直接将个人、组织或项目的页面托管于 GitHub 库或仓库 (repository) 中。
  • Hugo 是一个用 Go 语言编写的静态站点生成器，优势为速度快。（还有jek）

1 安装hugo 

sudo apt-get install hugo
# 或
sudo snap install hugo

# 测试安装
hugo version

2 本地创建 

# 创建
cd ~
hugo new site 【project-name】
cd ~/project-name

# 安装主题. 可以在github搜索hugo theme. 找适合自己的, 这里选用Stack.
cd ~/project-name/theme
git submodule add https://github.com/CaiJimmy/hugo-theme-stack theme/Stack

修改配置文件hugo.toml

baseURL = 'https://【github用户名】.github.io/'
languageCode = 'zh-cn'
title = 'MY Site'
theme = 'Stack'

# 本地预览
hugo server

# 网页输入
http://localhost:1313/

# 构建Hugo网站，将静态站点保存到public下
hugo

3 上传到github

这里选择使用两个仓库

  • 一个仓库储存整个代码, 设置为private. 并把/public文件夹添加到.gitignore
  • 第二个仓库储存public文件夹下的静态代码. 仓库名需要为 github用户名.github.io

3.1 第一个仓库

  • 在自己的github下新建repo. 仓库名随意，应为【project-name】
cd ~/project-name
git init

# 添加.gitignore
cd ~/project-name
touch .gitignore
echo "/public" > .gitignore

git add .
git commit -m "repo init"
git branch -M main
git remote add origin git@github.com:【github用户名】/【仓库名】.git
git push -u origin main
  • 第二个仓库
cd ~/project-name/public/
git init
git add .
git commit -m "page first commit"
git branch -M main
git remote add origin git@github.com:【github用户名】/【github用户名.github.io】.git
git push -u origin main
